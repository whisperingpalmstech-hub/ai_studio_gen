NOTEBOOKLM-STYLE GENERATION — COMPLETE GEMINI API CALL STRUCTURE
File: src/services/notebookLMSlideGenerator.ts

OVERVIEW
--------
The NotebookLM generation pipeline makes 5 types of Gemini API calls across 4 stages.
Text calls go DIRECTLY from the browser using VITE_GEMINI_API_KEY (client-side env var).
Image calls go via Supabase Edge Function (gemini-ebook-image-generator) to keep
the image API key server-side and avoid CORS.

BASE CONSTANTS
--------------
GEMINI_API_BASE     = https://generativelanguage.googleapis.com/v1beta/models
GEMINI_TEXT_MODEL   = gemini-3-pro-preview
GEMINI_IMAGE_MODEL  = gemini-3-pro-image-preview
TEXT_TEMPERATURE    = 0.1   (very low — grounded, factual)
IMAGE_TEMPERATURE   = 0.3   (slightly higher for visual creativity)

ENTRY POINT (AdaptiveCurriculumMaker.tsx)
-----------------------------------------
generateNotebookLMForChapter(chapter, subjectName, onProgress)
  1. formatChapterForNotebookLM(chapter)
       Converts BookletChapter into a markdown string:
       # Chapter Title
       ## Introduction  ...text...
       ## Section Title ...text...
       ### Examples:    - example 1
       ### Key Points:  - key point 1
       ## Summary       ...text...
       ## Practice Questions  1. question...

  2. Calls generateNotebookLMSlides(request, onProgress)
       request = {
         lectureTitle:       chapter.chapterTitle,
         lectureContent:     formatted markdown string,
         targetSlides:       enhancedModeOptions.slidesPerChapter,  (default 10)
         audience:           'intermediate',
         theme:              NOTEBOOKLM_THEMES[enhancedModeOptions.theme],
         includeInfographics: enhancedModeOptions.generateSlides,
         includeMindMap:      enhancedModeOptions.generateMindMaps,
         includeFlashcards:   enhancedModeOptions.generateFlashcards
       }

## Management
First-line: ACE inhibitors, Beta-blockers, MRAs, SGLT2 inhibitors (HFrEF).

Heart failure management requires neurohormonal blockade and device therapy.

================================================================================
FINAL RETURN VALUE  —  NotebookLMResponse
================================================================================
{
  slides: NotebookLMSlide[],     // N slides, each with imageUrl (base64 png or svg)
  mindMap: DiagramData,          // nodes + connections from Call 4
  flashcards: NotebookLMFlashcard[],  // 15-25 cards from Call 5
  studyGuide: StudyGuide,        // learning objectives + practice Q from Call 6
  summary: string,               // executiveSummary from Stage 1
  generatedAt: string,           // ISO timestamp
  theme: SlideTheme,             // the theme object used throughout
  stats: {
    totalSlides: number,         // slides.length
    slideTypes: Record<string, number>,  // { "title": 1, "infographic": 3, ... }
    flashcardCount: number,      // flashcards.length
    estimatedDuration: string    // slides.length * 2.5 minutes
  }
}

This object gets attached to chapter as:  chapter.notebookLMContent = result
Then used in PDF export, in-browser preview, and download functionality.

================================================================================
PROGRESS CALLBACKS  (passed through to UI)
================================================================================
Stage 1 complete  →  "Content analyzed successfully"       20%
Stage 2 complete  →  "Slide layouts created"               40%
Per-image done    →  "Generating slide N/total"            45-70%
Stage 4 complete  →  "Study materials complete"            90%
All done          →  "Generation complete!"               100%

================================================================================
ERROR HANDLING
================================================================================
- If Stage 1 JSON parse fails  → throws "Failed to analyze content structure"
- If Stage 2 JSON parse fails  → throws "Failed to generate slide content structure"
- If image generation fails    → falls back to SVG (no error thrown, continues)
- If mind map fails            → returns undefined (non-fatal)
- If flashcards fail           → returns [] (non-fatal)
- If study guide fails         → returns undefined (non-fatal)
- If entire NotebookLM fails   → toast shown, returns null, chapter saved without enhanced content
